## 1. 项目定位

WUCO Proactive Agent 不是"主动提醒工具"，而是一个**基于 context、memory、行为轨迹和任务状态的主动服务层**。它不等用户明确下指令，而是在用户搜索、写作、翻译、生图、文档总结等过程中，实时识别潜在需求，并以**可控、低打扰、可解释**的方式提供下一步服务。

一句话：让 AI 发现用户正**卡住、遗漏或即将需要帮助**的时刻，提前给出可执行服务。

**核心创新点不在"主动"本身，而在对主动性的治理**——触发评分 + 每次触达可解释 + 用户分级控权。这正面回答评委必问的"主动服务会不会变成骚扰"。

### 不做什么（明确排除）
- 不做"每天提醒喝水/看天气/写日程"式的智能提醒。
- 不做"读取用户所有数据随时推送"式的无差别打扰。
- 不在医疗、财务、情感、隐私文件等敏感场景主动推送（只被动响应）。

## 2. 评审对齐

赛题评审核心：**场景逻辑的合理性验证（重点验证主动服务的设计逻辑与用户体验）**。
本作品的所有设计决策都服务于这条：
- "设计逻辑" → 用**可见的感知面板 + 触发评分模型**，把预判从"魔术"变成"看得见的逻辑"。
- "用户体验" → 低打扰默认、可解释卡片、一键关闭/撤回、主动程度可调。

---

## 3. 核心机制（5 层）

| 层 | 职责 | 关键设计 |
|---|---|---|
| 1. Context 感知层 | 识别用户当前在做什么 | 信号源：当前输入 / 最近对话 / 上传文件 / 时间信息 / 用户偏好 / 任务状态 |
| 2. 需求预判层 | 判断用户真正想完成的任务 | **识别意图爬升轨迹**（"是什么"→"怎么做"→"截止时间"），而非数搜索次数；次数仅作辅助信号 |
| 3. 触达判断层 | 是否/何时/以何形式介入 | 触发评分模型（见 §4），过阈值才介入 |
| 4. 主动触达层 | 承载并呈现建议 | 低/中/高三级打扰；**每张卡片带"💡为什么提示我"可解释** |
| 5. 反馈学习层 | 用反馈反向优化 | 接受/忽略/拒绝/别再提醒 → 调权（Demo 中以**可见微动效**呈现，不做真实学习器） |

### 需求预判：意图爬升轨迹（重点）
不靠"连续 3 次"这种计数，而靠**问题语义的升级方向**：
- 阶段一·认知："X 是什么 / 有哪些方向"
- 阶段二·方法："怎么做 / 怎么规划 / 给个框架"
- 阶段三·行动："截止时间 / 提交要求 / 帮我写出来"
当 AI 检测到提问从认知 → 方法 → 行动爬升，判定用户进入"复杂任务准备阶段"，主动补位。

---

## 4. 触发评分模型

```
触发分 = 需求强度 × 时效性 × 置信度 × 用户接受度 − 打扰成本
```

| 因素 | 判断标准 |
|---|---|
| 需求强度 | 用户是否连续围绕同一任务操作、意图是否在爬升 |
| 时效性 | 是否检测到截止日期 / 会议 / 提交节点 |
| 置信度 | AI 能否明确判断"下一步"是什么 |
| 用户接受度 | 用户过去是否接受过类似主动建议 |
| 打扰成本 | 是否正在高频输入、是否刚拒绝过同类建议 |

仅当 `触发分 ≥ 阈值` 才主动提示。**该评分在 Demo 的"感知面板"中实时可见**，是作品差异化的核心可视化资产。

## 5. 场景设计

### 5.1 主线深场景（完整闭环）：研究/写作型任务的主动推进
**用户故事**：一位用户用元宝准备一次重要的工作汇报 / 调研报告。行为序列：
1. 搜索"竞品主动式 AI 有哪些做法"（认知）
2. 搜索"汇报方案怎么搭框架"（方法）
3. 上传一份活动/需求说明文档
4. 问"提交截止时间是什么"（行动）

传统 chatbot 只逐条回答；WUCO Proactive Agent 识别出"用户在准备一个复杂任务"，在恰当时机主动提供：
- 生成方案/汇报框架
- 从文档提取待办与截止时间
- 设置进度提醒
- 复用用户既有写作风格

**彩蛋收尾**：演示末尾点明——"这套参赛方案本身，就是它这样帮我推进出来的"，自指增强记忆点（作为彩蛋而非主线，避免"为演示而演示"）。

### 5.2 静态概念场景 A：文档总结 → 识别待办与风险
用户总结/上传文档，文中含截止时间、提交要求、评审标准、任务清单。
主动服务："这份文档里有 4 个关键待办和 1 个截止时间，是否生成提交清单和时间规划？"
价值：从"总结内容"升级为"帮助完成任务"。

### 5.3 静态概念场景 B：写作/翻译 → 风格一致性复用
用户多次让元宝润色/翻译同类内容。
主动服务："我注意到你偏好'口语化、精简、逻辑清晰'，是否将本次改写自动套用该风格？"
价值：memory 不只记住用户，而是主动复用偏好，减少重复指令。

> 范围策略：先把 5.1 做到极致闭环，5.2/5.3 以静态概念卡呈现；**若进度有余，再将其升级为可交互完整场景**。

## 6. Demo 形态与技术栈

- **形态**：可独立运行的纯前端网页 Demo（双击 `index.html` 即可运行，无需后端/安装）。
- **技术栈**：HTML + CSS + 原生 JS（或极轻量无构建依赖）。一个脚本化"模拟引擎"驱动主动逻辑：维护对话状态、计算触发分、按规则弹出卡片。
- **跨会话主动的演示**：提供"⏩ 时间快进"控件，让"AI 隔几天带着新信息回来找你"可当场演出，无需真实等待。
- **配套交付**：一份参赛方案说明文档（提交材料）。

---

## 7. 界面结构（4 界面）

1. **元宝对话页**：保留聊天体验 + 主动服务卡片 + 任务状态识别标签 + 下一步建议按钮。
2. **主动服务卡片**：结构为「我发现你可能正在… / 我可以帮你… / 是否现在处理？」+ 按钮（立即生成 / 稍后提醒 / 不再提示此类）+ "💡为什么提示我"。
3. **任务面板**：展示 AI 自动整理出的任务及状态（已完成/进行中/未开始）。
4. **主动服务设置页**：主动程度（关闭/低/标准/积极）、允许使用的上下文（当前对话/历史/文件/日程）、触达方式、敏感内容默认不分析、一键清除记忆。
5. **感知面板（新增·核心差异化）**：实时显示当前捕捉到的信号（主题连续性、意图阶段、检测到的截止日…）与算出的**触发分**，把"预判"可视化为"设计逻辑"。

## 8. Demo 演示脚本（≤4 分钟，5 步 + 彩蛋）

| 步骤 | 用户动作 | 系统表现 | 感知面板 |
|---|---|---|---|
| 1 | 输入认知型问题 | 正常回答 | 主题=新；意图=认知；触发分 低 |
| 2 | 输入方法型问题（同主题） | 正常回答 | 主题连续性↑；意图=方法；触发分 升 |
| 3 | 上传说明文档 + 问截止时间 | 后台识别任务意图 | 检测到截止日；置信度↑；触发分 过阈值 |
| 4 | —（AI 主动）| 弹出主动卡片："发现你在准备一个复杂任务，是否整理成方案框架？"含"💡为什么" | 触达发生，记录"已接受" |
| 5 | 点击"生成方案框架" | 生成：项目定位/场景/触发机制/Demo脚本/提交待办；任务面板同步填充 | 接受 → 该类建议权重↑（微动效）|
| 彩蛋 | 点击"⏩时间快进" | "距提交截止还有 N 天，Demo 主流程尚未完成，是否现在补齐？" + 自指点题 | 时效性触发高打扰提醒 |

## 9. 隐私与体验平衡机制（评委必看）

1. **默认低打扰**：主动服务默认仅在对话内出现，不发系统通知。
2. **权限分级**：当前会话 / 历史记忆 / 外部数据（文件·日程）三级授权。
3. **每次触达可解释**：卡片可展开"为什么提示我"，如"因为你连续围绕同一主题提问并上传了说明文档"。
4. **可关闭、可撤回**：每张卡片含 不感兴趣 / 本次关闭 / 不再提示此类 / 管理记忆。
5. **敏感场景不主动**：医疗、财务、情感、隐私文件只被动询问，不主动推送。

---

## 10. 作品核心亮点（汇报话术）

1. 不是提醒工具，而是**任务流 Agent**——嵌入搜索/写作/总结/生图等元宝高频场景。
2. 不是乱推送，而是**机会点触发**——需求强度×时效性×置信度×接受度−打扰成本。
3. 不是替用户决定，而是**给用户选择**——只建议不强制。
4. 不是一次性功能，而是**可学习机制**——接受/忽略/拒绝反向调权。
5. 不是牺牲隐私换智能——**权限分级 + 触达解释 + 主动程度 + 记忆管理**。

## 12. 范围与风险

**范围（个人赛可完成）**：主线场景完整闭环 + 感知面板 + 4 界面（设置页够用即可）+ 时间快进 + 反馈微动效 + 2 静态概念场景（余力升级）。

**已识别风险与对策**：
- 预判被质疑"写死" → 感知面板把信号与评分公开，呈现为设计逻辑。
- wow 押在单张卡片、方差大 → 文案极度克制真实、宁缺毋滥、带"为什么"。
- 自指演示显得做作 → 降为彩蛋，主线用真实用户场景。
- 范围烂尾 → 先深后广，反馈学习只做动效。
- 跨会话主动难演 → 时间快进控件。
- UX 直接影响评分 → 省下的时间全部投入界面打磨。
